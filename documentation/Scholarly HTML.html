Introduction

Visual impairment is a decreased ability to see to a degree that causes problems not fixable by usual means, such as glasses. Some also include those who have a decreased ability to see because they do not have access to glasses or contact lenses. The term blindness is used for complete or nearly complete vision loss. Visual impairment may cause people difficulties with normal daily activities such as driving, reading, socializing, and walking.

This project helps people with visual problems by providing a browser extension that allows them to better distinguish the content of a web page. Using this extension, a user have the ability to change the color scheme and rezise the content of a page and also, the ability to listen the content.

Project identity

The identity of this project, which is Visia extension, was developed by following the steps/stages of a branding process. The branding process is important before creating the application for more reasons. First of all, we need to know our clientsâ€™ needs and how we can satisfy their needs with our product or service (in this case our application). Second of all, branding helps us finding the best way to present the application to our target audience in order to make it interesting and useful, this means we will use the most suitable visual and verbal identity to communicate with them.

The branding steps in creating the application identity are the following:

  • Research our target audience: in this case people who have vision problems (developing their profiles: demographic, social, psychographic characteristics)
  • Developing the conceptual identity (the values and concepts that define our application)
  • Developing the visual and verbal identity of the application (name, logo, slogan, tagline)

The visual identity (name and logo) expresses the main concepts of the application. We can see that both the name (VISIA) and logo (the eyes) are related to the word vision which is the main problem that the application is meant to improve. The details in the logo can be also explained: the eye is not complete (a part of it is missing) in order to express the lack of good vision in people who are going to use this application. The name is simple and short in order to be more easily memorized by the target audience.

Interaction design patterns

3.1 Contrasting Colours for Better Readability

Contrast is the difference between the lightest color (e.g. the background), and the darkest color (e.g. the text).

Therefore, to ensure text is readable by people with visual impairments, the WCAG has provided a contrast ratio guideline. They suggest that a contrast ratio of at least 4.5:1 should exist between a text and its background. This ratio drops to 3:1 for larger text (24px or 29px bold).

I choose to use the "#32796D" color for background and "#DDFFEE" for text (the values from the example).

3.2 Font Selection

When we design an application for people with visual impairments we need to avoid complicated, decorative fonts, and reserve such styles, for emphasis only. The goal is to use easily recognizable characters such as Arial, Verdana, Tahoma and Sans Serif versions of any typeface are often more legible, since the letters are simple.

Another important aspect is the font size. The larger the font, the easier it is for visually impaired people to read

Verdana font and a font size of 14 points is used for every text in the extension.

3.3 Text Alignment

It is recomended that the text to be left aligned rather than fully justified, center or right aligned. Fully justified text presents particular difficulties for readers with dyslexia or visual impairments.

3.4 Navigation

The extension is build to be easy to perform any task. The main menu ( floating menu ) can be opened using "ctrl" + "Y" keys or by left clicking on the extension icon. The menu can be moved around the screen.

Another type of menu that was used is the drop-down menu that can be opened by right clicking on the extension icon. For options menu I choose to use a modal window because there are multiple options and the font size needs to be bigger in order to be easy to distinguish beetween them.

Implementation details

In order to create a extension we need to define a file called "manifest.json". In this file we can set the name of the extension, add a icon, define which html document will pop when we click on the extension icon and define the option page. Also, here we define a list of javascript files which have permissions to modify and extract data from the current page.

Next step was to create the html file that will pop-up when we click on the extension icon. We have a simple page that contains icons for play, stop, resume text reading and two inputs from which we can select the colors that will be applied on the elements from DOOM.

Options page contains two main categories: settings for reading and settings for color. In this html document we imported a javascript file that store and read from the browser store the options provided by the user.

Content script is a javascript file that has permissions to modify and to extract data from the current page. This can communicate with the script, imported by the main html file, using sendMessage functionality provided by "chrome.tabs". When an action is triggered from the main html file a message is sent to content script that execute the desired action.

Web Accessibility Evaluation

In this chapter we will present only the techniques applied for the current project.

5.1 Text equivalent for every non-text element

All image elements should have a "alt" attribute defined and also a description ("longdesc" attribute).

In the extension we have only five images that are representet by the icons used in the main pop-up modal. All icons have an "alt" and a "longdesc" attribute in which we present what action is triggered when we click on them. File provided in "longdesc" is in resources folder and is called description.txt.

5.2 Information conveyed with color is also available without color

Color brightness is determined by the following formula: ((Red value X 299) + (Green value X 587) + (Blue value X 114)) / 1000. The brightness in our case is 250.036

Color difference is determined by the following formula: (maximum (Red value 1, Red value 2) - minimum (Red value 1, Red value 2)) + (maximum (Green value 1, Green value 2) - minimum (Green value 1, Green value 2)) + (maximum (Blue value 1, Blue value 2) - minimum (Blue value 1, Blue value 2)). In our case the value is 434.

5.3 Check document for use of style sheets

Every html file has a css file that is imported in the head component. Style embeded in html wasn't been used.

5.4 Check document for relative units of measure

All the div's defined in the css files are using "%". Only for icons images "px" is used, but this is allowed for this type of elements.

5.5 Check document for header nesting

Header levels must not increase by more than 1 level. We used only h1 and h2 in this extension, the rule is respected.

5.6 Verify that header elements are not used for formatting

A header element's text content can have maximum 20 words. The longest header is "Color of background".

5.7 Check for valid default values of INPUT, TEXTAREA, and SELECT elements

Input elements that have a type of "text", "checkbox" or "radio" must have at least one word of text in their "value" attribute. All inputs have a default value. For example, in the options page the default values for voice setting are "US English Female" for voice, 1 for pitch, 1 for volume and 1 for rate. For color settings all checkboxes are enabled.

User manual

6.1 Main page (popup page)

Main page can be accessed by left clicking on the extension icon from the top-right corner of the browser

This page contains the following elements:

  • Play button - selected text will be played in audio format
  • Pause button - will pause the reading of text
  • Stop button - will stop the reading of text
  • Zoom out button - the page content will be zoomed out by 50 percentage
  • Zoom input - the page zoom will be set to provided value
  • Zoom in button - the page content will be zoomed in by 50 percentage
  • Change text color - will change the text color for elements selected in options page
  • Change background color- will change the background color for elements selected in options page

When we click on an input for change color a modal will open in which we can select the color that will be applied.

6.2 Options page

Options page can be accessed from the extension menu. To open the menu we need to right-click on the extension icon. After this we need to select Options. A new tab will be displayed.

6.2 Options page

Options page contains two main categoryes: options for voice and options for color. When we click on one of the options a new component will pe expanded inn which we can see all the propertyes that we can change.

Voice settings are: voice type, pitch, volume and rate. We can change the values and the changes will be applied to the main page.

In the color settings we can select for which html elements the coloring will be applyed. We have multiple checkboxes for text and background. If we select "All elements" all the other checkboxes will become active, or will be disable, it depends of the initial value. If "All elements" is checked and we unchecked one of the propertyes, "All elements" will becoma also unchecked.
